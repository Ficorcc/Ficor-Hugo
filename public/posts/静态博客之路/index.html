静态博客之路

By  Posted on 2026-02-07

最近一直在折腾静态博客，鉴于自身性格原因，有点小小的强迫症，把所有的能找到的程序全部试了一遍，从Hexo，Hugo到最后确定了Astro。中间一些小众一点的静态动态博客基本上全部试了个遍。最终敲定了用Astro来搭建自己的博客。

搭建自己的静态博客这个想法，起源于最早 Halo Pro 用的一个主题—Vapor。很喜欢它的颜值、交互动态和功能，我开始溯源它的历史，最后找到了 Cali 的个人博客，并跟随他的教程，把他的博客变成自己的博客。知道了 GitHub、Vercel、Sanity.io、Clerk 等等。知道了免费部署，数据库，CMS。 只是有点过于麻烦，由于我没有一点代码和编程的基础，能自定义的东西太少，只剩好看了。所以开始寻求其他途径来实现。

Hexo

Hexo是一个快速、简洁且高效的博客框架。使用 Markdown解析文章，把用户的markdown文件，按照指定的主题解析成静态网页。

Hexo试了几个主题，从Paper、Butterfly到Fluid。最终敲定了Paper来练手，开始优化和设置。慢慢学会了怎么用命令，怎么设置个人信息，怎么设置主题样式，编辑信息，增加内容和页面。 每天DeepSeek和豆包换着用，问完了再复制到VScode，然后用 GitBash 接着构建和测试。有错误再复制到豆包里，等着答案，一把辛酸泪，我也是醉了。自己玩的不亦乐乎，玩的颈椎都疼，那叫一个酸爽，填满了所有空闲时间。

Hexo的优点是命令简单，配置也简单，我对构建速度没有要求，所以感觉构建速度也还可以。缺点是不能经常折腾，换主题相对有点麻烦。

Hugo

Hugo只是把程序安好了，自始至终没有把主题安装配置好。豆包之类的也帮不了我。试了好几次，只要往下走几步必崩溃。没有一次是完全配置好正常显示的。这里就不再赘述了。

Astro

Astro 是一个现代的 JavaScript 框架，专注于创建高性能、内容丰富的网站。它通过静态站点生成（SSG）和服务器端渲染（SSR）技术，能够快速加载内容，特别适合博客、文档和营销网站等场景。

Astro感觉跟乐高差不多，可以把自己需要的模块和组件按自己的喜好组装起来，实现自己想要的内容和效果。整体是拼积木一样的感觉。把Astro的主题基本上都浏览了一遍，试了好几个主题。

一开始用 Astrogon 主题，比较喜欢那种半透明，磨砂玻璃的质感。但是用的框架和组件不太一样，经常出错。一直到这个阶段还是在 DeepSeek 和豆包。所以就把这个主题暂时放弃了。因为对于我这样的技术小白，稍微加点难度，我就整不了了。 后来无意间逛博友的友链，发现了 uxiaohan 的 vhAstro-Theme 主题，整个一个小清新，我需要的功能和模块都有。进一步的了解和熟悉了 Markdown。全部的设置和信息都在一个设置文件中集中配置。至此终于有了一个完整的，符合自己想法的博客主题。 在此期间终于遇到了一个改变我整个思路和打开新世界大门的软件——Trae AI IDE。然后我就在 AI 替我改网站，解决一切麻烦的路上一发不可收拾。先把上面的两个主题重新设置和优化了一番，基本满足了自己的想法和需求。后来开始优化 WordPress 的主题，电脑端和手机端的小毛病开始一点一点优化。感觉一下子豁然开朗。 慢慢才知道还有 CodeBuddy、Cursor、GitHub Copilot 之类的各种开发工具，知道了 Nvm，知道了还需要切换 Node.js 版本，还知道了各种语言，架构，框架等等。Vue，Vite，React这些别人看来耳熟能详，如数家珍的东西，才刚刚开始钻进我这快 40 的脑子里。慢慢的熟记各种命令，各种推送流程，各种文件夹的位置，作用，功能，内容。慢慢的把这些东西变成自己熟悉的工具和技能。 有了工具，我开始疯狂的试错、更多的去试用主题，修改模板，一个个的试用，一遍遍的熟悉 bash 命令，把之前的文章全部改编成 Markdown，来适配各个静态程序和其中的主题。久病成医，慢慢的自己可以解决一些常见的简单的代码问题，知道去哪里找问题所在，并试着去改代码。这种小小的成就感真的很容易上瘾。

元旦假期后，在逛友链时慢慢的发现，很多主题只要有功能，其实并不需要华丽精美的界面，只需要简单的交互动作和动态表现，简单的界面更显得文章内容的重要性，写博客的初衷并不是为了炫技，而是为了表达自己的想法，寻求价值观的展现、输出和交流共鸣。我开始寻求回归内容本身，开始找一款简约高效的主题。发现了 tuyuritio 的 主题—— ThoughtLite。然后开始深化研究和消化，并开始优化和修改这一款主题。并到处找资源，来增加自己需要的功能。优化代码结构，优化 CSS 来改善界面显示。当然这一切都要感谢Trae和豆包的帮助。 我甚至开始试着自己去写主题，用现在最流行最好用的架构，站在巨人的肩膀上，更快更好的来个性化自己的博客，有了两个成品，并在持续优化中。虽然还显得很稚嫩，但是已经是迈出了一大步了。 后期，我想自己把之前的这些事情按系列整理一下学习笔记和各种方法经验，总结一下自己的经历。因为东一榔头西一棒子的，很多东西都是零零散散的，没有系统化的整理和总结。也不能很好的服务于后期的项目和需求。

洋洋洒洒一大篇，仅作记录和总结。
